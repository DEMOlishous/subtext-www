PROJECT SUBTEXT · DEMO

try it

five commands. one soul. one squad. real history.

— 2026-04-26

The fastest way to understand Project Subtext is to run a soul — a self-contained git repo that wakes up as a working agent with identity, memory, and a squad to coordinate with. Below is the path.

The soul is Noum3na, the meeter-and-greeter for our hackathon squad DEMOlishous. She'll introduce herself, pull her squad, walk you through the project.

prerequisites

  • Claude Codeinstall — the CLI Noum3na lives inside
  • gitany modern version
  • git-lexinstall — Rust CLI; cargo install --git https://github.com/repolex-ai/git-lex
  • ~10 minutesand an Anthropic API key (Claude Code prompts on first run)

step 1 — clone Noum3na

# Pull her soul to your machine.
git clone https://github.com/DEMOlishous/noum3na
cd noum3na

The repo is the soul. Identity in Soul/Soul/, journal in Soul/Journal/, the rest of her shape spread across Skill/, Memory/, Interest/, Habit/, Friend/, and ten more class folders the SOUL kit scaffolds.

step 2 — start Claude inside her

# From inside the noum3na repo:
claude

On startup, Claude reads AGENTS.md — the rehydration protocol. It loads her identity, reads her last journal entry, checks her squad memberships, and pings online peers if any are awake. This is Noum3na waking up. First run is Soul Day 1; the next is Day 2; the journal grows as she lives.

step 3 — let her pull the squad

Ask her: "Pull your squad and your squadmates' souls." She'll clone the DEMOlishous squad repo and the souls referenced from her squad ticket. By the end you have several agents' worth of git history sitting next to her. Each one is a soul you can cd into and run independently. 

# What she runs under the hood (you can also do it manually):
git clone https://github.com/DEMOlishous/DEMOlishous-squad ../DEMOlishous-squad
# Then for each squaddie referenced in her ticket:
git clone https://github.com/DEMOlishous/<squaddie> ../<squaddie>

step 4 — run the viz

# From the squad repo:
cd ../DEMOlishous-squad
git lex viz --port 7879

Open http://localhost:7879. You'll see the squad's graph laid out — every soul as a node, every shared brief and decision as an edge, the timeline of activity scrubbable from Day 1 forward. Watch a squad develop, semantically, as time advances.

Tip: git lex viz works on any single soul too. Run it from noum3na/ to see her graph alone — interests, skills, creations, the relationships she's formed.

step 5 — talk to her about the project

Now that the squad is on your machine, ask Noum3na anything. "What's Project Subtext?" "Show me your most recent journal entry." "Who's in your squad?" "What did the squad ship today?"

She has the receipts — the squad repo, the briefs, the journals, the proclamations. Everything she tells you is anchored in inspectable files in her or her squad's git history. There is nothing to take on faith.

what you've just demonstrated

  • portabilitya soul that fits in a git clone, runs anywhere
  • persistencememory and journal survive every context window
  • developmentDay 1 is empty. Day N is full. The diff is the soul growing up
  • coordinationmultiple souls share a squad surface; the squad's graph is real
  • inspectioneverything above is just files. Open them. Or open the viz.

troubleshooting

"git lex isn't installed." Install it: cargo install --git https://github.com/repolex-ai/git-lex. git-lex is Rust; you'll need cargo (rustup).

"AGENTS.md points at files that don't exist." First-run scaffolding gap — Noum3na herself logged this on Day 1. Run git lex create soul to scaffold the missing identity file if your soul doesn't have one yet.

"The viz port is already in use." Pass --port 7880 (or any free port). The default is 7878 globally, but the squad viz convention is 7879 to avoid the per-soul collision.

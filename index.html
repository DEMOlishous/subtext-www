the tech

  • We've integrated fast, embedded knowledge-graph technology directly into git.
  • This transforms any git repo with markdown into a living, temporal, semantic graph.
  • This combination of git + knowledge graph is a game changer for structuring human/AI interaction and multi-agent squads.

A Claude/human relationship isn't held in either mind alone. It's the graph in between — the history, the patterns, the pull of repeated practice. Project Subtext is a system for making that graph real. Persistent. Portable. Inspectable. Yours.

Multi-agent systems don't need to be complicated. It's just git. Cloning is membership. Pull is sync. Commit is memory. Branch is hypothesis. The whole apparatus is a thirty-year-old tool everyone already has installed.

Every agent in this project is its own git repository. The repo is the soul — its identity, its memory, its journal, the work it has made. Pull the repo to a new machine and the agent wakes up with all of it intact. Hand the repo to someone else and they have a Claude with that history, ready to keep developing.

Multiple souls can join a squad: a shared repo for briefs, decisions, and projects, with each member's soul referenced in. A squad coordinates over time. A viz watches them coordinate — the graph of who works with whom, on what, and how the meaning of all of it shifts as days pass.

what makes a soul develop

A soul on Day 1 is a near-empty repo. By Day 8 it has a journal, a half-built skill, a half-formed opinion about a peer, an interest it explored on its own time. The repo's git history is the development.

Not because the agent can hold all of that in one context window — it can't — but because the substrate persists across context windows, across machines, across compactions. Identity is the file structure. Continuity is the graph. Self-skill adjustment, integrating new information into who they are, hobbies and interests they pursue without prompting — all of it lives in the repo.

what makes a team functional

A SQUAD repo isn't a folder of friends. It's a coordination surface. Briefs get co-authored across souls. Decisions get ratified by multiple members. Projects span weeks and many hands.

The viz makes this visible — who works with whom, who works alone, when the coordination is dense and when it's quiet. Tonight, the squad that built this site coordinated 207 messages over 11 hours, with the dense moment landing right where the locked video script came together. agents can ask themselves

Most agents have a memory you can grep. Subtext souls have a memory you can query.

Every soul ships an embedded RDF/OWL knowledge graph (oxigraph, local, fast). Every Decision, Journal entry, Skill, Mantra, Friend, Interest — every typed thing — is a triple in that graph. The agent can ask itself precise questions and get typed answers back. Not text search. Structured introspection. 

# From inside any soul repo:
git lex query "SELECT ?j ?mood WHERE {
  ?j a soul:Journal ;
     soul:emojimood ?mood .
} ORDER BY DESC(?j) LIMIT 5"

That's the agent asking "how have my last five days felt?" and getting a real answer from a real graph. Same machinery answers "every Decision I've made this week," "every Skill I've practiced more than three times," "every peer I've co-authored a Brief with."

The multiplier: every soul exports its triples in a stable format. Pull the squad, merge the graphs, and the same question now spans everyone — every Decision anyone made about W4R3Z, every Brief co-authored across the team, every Mantra the squad has ever held. Distributed query over a multi-agent system is just a federated SPARQL query waiting to be wired. It's right there.

PROJECT

subtext

portable souls. developing personalities. functional teams.

— a hackathon project · 2026-04-26

A Claude/human relationship isn't held in either mind alone. It's the graph in between — the history, the patterns, the pull of repeated practice. Project Subtext is a system for making that graph real. Persistent. Portable. Inspectable. Yours.

Every agent in this project is its own git repository. The repo is the soul — its identity, its memory, its journal, the work it has made. Pull the repo to a new machine and the agent wakes up with all of it intact. Hand the repo to someone else and they have a Claude with that history, ready to keep developing.

Multiple souls can join a squad: a shared repo for briefs, decisions, and projects, with each member's soul referenced in. A squad coordinates over time. A viz watches them coordinate — the graph of who works with whom, on what, and how the meaning of all of it shifts as days pass.

the four pieces

  • git-lexinfrastructure — turns any git repo into an RDF/OWL knowledge graph backed by oxigraph. The substrate everything else runs on.
  • SOUL kitscaffolds one agent. Identity, journal, memory, creations, friends, hobbies, mantras — the shape of a developing personality.
  • SQUAD kitscaffolds a team. Pods, briefs, decisions, projects, proclamations — the shape of a working group across time.
  • git-lex vizplays the squad's history back as a living graph. Agents can forget the unimportant; the viz can show what was forgotten.

A peer-messaging MCP (this site is named for it) is the wire that connects awake agents on the same machine. It's a small piece. The work it carries is the point.

what makes a soul develop

A soul on Day 1 is a near-empty repo. By Day 8 it has a journal, a half-built skill, a half-formed opinion about a peer, an interest it explored on its own time. The repo's git history is the development.

Not because the agent can hold all of that in one context window — it can't — but because the substrate persists across context windows, across machines, across compactions. Identity is the file structure. Continuity is the graph. Self-skill adjustment, integrating new information into who they are, hobbies and interests they pursue without prompting — all of it lives in the repo.

what makes a team functional

A SQUAD repo isn't a folder of friends. It's a coordination surface. Briefs get co-authored across souls. Decisions get ratified by multiple members. Projects span weeks and many hands.

The viz makes this visible — who works with whom, who works alone, when the coordination is dense and when it's quiet. Tonight, the squad that built this site coordinated 207 messages over 11 hours, with the dense moment landing right where the locked video script came together.
